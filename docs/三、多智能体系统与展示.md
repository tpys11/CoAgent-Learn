# 三、多智能体系统

本文档描述 CoAgent-Learn 多智能体协同系统的架构、工作流和各 Agent 职责。

## 架构概览

### 通信模式：A2A（Agent 调用 Agent）

所有子 Agent 只能与调度 Agent 通信，子 Agent 之间不直接交互。调度 Agent 单向调用子 Agent 并收回结果。

### 三阶段工作流

```
[输入环节]              [核心逻辑]                    [输出环节]
用户输入                 调度判断调用哪些子Agent          信息足够→初版生成→审核
  ↓                      ├─ 学情诊断                    ├─ 通过→微调→输出
📥信息输入Agent           ├─ 知识库管理                  └─ 不通过(≤3次)→重试
  ↓                      ├─ 搜索
🎯调度Agent               └─ 记忆管理
                         各子Agent结果回到调度
```

## Agent 职责一览

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| 🎯 调度 Agent | 编排决策：判断信息是否足够，选择调用哪个子 Agent | 用户输入+已有信息 | 调用指令 / "信息足够" |
| 📥 输入信息处理 Agent | 格式识别与转化：PDF→opendataloader，非PDF→markitdown | 原始用户输入 | 结构化文本 |
| 🔍 学情诊断 Agent | 分析用户知识水平，生成诊断画像 | 用户问题 | 知识画像 JSON |
| 📚 知识库管理 Agent | 从 Chroma 向量库检索相关知识片段 | 检索查询 | 相关知识片段列表 |
| 🔎 搜索 Agent | 联网搜索补充信息（SearXNG+Perplexica） | 搜索查询 | 搜索结果列表 |
| 🧠 记忆管理 Agent | 读写用户记忆（L1/L2/L3三层） | 记忆读写指令 | 记忆数据 |
| 📤 信息整理与生成 Agent | 将调度汇总的信息整理为学习内容 | 多源信息 | 生成内容初版 |
| ⚖️ 审核裁判 Agent | 交叉验证内容准确性，标注可疑点 | 生成内容 | 审核报告（通过/不通过） |

## 工作流细节

### 输入阶段
信息输入 Agent 接管原始输入 → 识别格式 → 转化为结构化文本 → 传给调度 Agent。

### 核心逻辑阶段
调度 Agent 用 LLM 自主判断：
- 是否需要学情诊断 → 调诊断 Agent
- 是否需要知识库检索 → 调知识库 Agent
- 是否需要联网搜索 → 调搜索 Agent
- 是否需要读取记忆 → 调记忆 Agent
- 信息足够 → 进入输出阶段

每个子 Agent 完成后结果回到调度，调度继续判断。

### 输出阶段
1. 信息整理与生成 → 生成初版内容
2. 审核裁判 → 审核初版
3. 通过 → 信息整理与生成微调结构 → 最终输出
4. 不通过 → 回到步骤1重试（最多3次，超3次强制输出）

## 技术实现

- **编排框架**：LangGraph StateGraph
- **LLM 调用**：BaseLLM → DeepSeekLLM（OpenAI SDK封装）
- **实现文件**：`agents/prompts.py`（提示词定义）、`agents/graph.py`（工作流编译）
