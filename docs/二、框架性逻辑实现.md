# 二、框架性逻辑实现

本阶段搭建了项目的两大底层能力：MVP 整体架构和路线图、大模型统一调用层。

---

## 一、MVP 系统架构与路线图

**目标**：2026年7月31日前，做出能跑通的 Web 应用。用户输入学情，系统通过三 Agent 协同生成个性化 AI 领域学习资源。

```
用户浏览器 ──→ React 前端（端口5173）
                    │
            FastAPI 后端（端口8000）
                    │
        ┌───────────┼───────────┐
        │           │           │
   学情诊断Agent  生成Agent  审核Agent
        │           │           │
        └───────────┼───────────┘
                    │
            LangGraph 编排器
                    │
        ┌───────────┼───────────┐
        │           │           │
    知识库RAG    用户画像存储   LLM调用
    (Chroma)     (SQLite)    (DeepSeek)
```

**约束**：单人开发，知识领域聚焦多智能体系统开发（Agent协同/LangGraph/向量检索/RAG），V1仅默认学习模式。

**明确不做**：知识图谱可视化、第二对话窗口、企业培训/应试模式、MCP 工具接入、用户登录/注册、流式输出。

**Agent 间数据流**（LangGraph State）：
```python
class AgentState(TypedDict):
    user_input: str
    profile: dict        # 诊断Agent写入
    generated: dict      # 生成Agent写入
    reviewed: dict       # 审核Agent写入
    retry_count: int     # 条件路由：幻觉>5%时重试
```

## 二、BaseLLM — 大模型统一调用层

在 `backend/core/` 下创建了统一的模型调用层，让上层 Agent 代码无需关心底层模型差异——调用两个方法即可。

### 新建文件

| 文件 | 职责 |
|------|------|
| `backend/core/__init__.py` | 包标记 |
| `backend/core/config.py` | 从 `.env` 读取全部配置，集中管理 |
| `backend/core/base_llm.py` | 基类 `BaseLLM` + DeepSeek 子类实现 |
| `tests/test_base_llm.py` | 独立验证脚本 |

### 配置统一入口

```python
class Config:
    DEEPSEEK_API_KEY / DEEPSEEK_BASE_URL
    LLM_MAX_CONCURRENCY / LLM_REQUEST_TIMEOUT
    SQLITE_PATH / REDIS_HOST / CHROMA_PERSIST_DIRECTORY
```

全项目一处导入 `from backend.core.config import config`。

### 核心接口

**`chat(messages)`** — 普通对话。内部自动：①过滤 DeepSeek R1 思考标签 ②超时重试（最多3次，间隔2s→4s→6s递增）③记录 token 消耗。

**`chat_with_json(messages, output_schema)`** — JSON 结构化输出（Agent 间传数据用）。自动注入 Schema 指令 + `response_format={"type": "json_object"}` + 两层容错解析（`json.loads` → 正则提取 → 抛异常）。

### DeepSeekLLM 子类

仅 6 行代码。新增模型只需写类似子类，Agent 层代码不改一行。

**技术选型理由**：使用 OpenAI SDK（非 langchain-deepseek）——DeepSeek/GLM/Qwen/Kimi 全部兼容 OpenAI chat/completions 接口，避免 langchain 版本冲突，JSON mode 依赖原生 SDK。

### 验证

```bash
cd guashuai-project
python tests/test_base_llm.py
```

有 Key 时执行两次调用并打印结果和 Token 消耗，无 Key 时安全跳过。
