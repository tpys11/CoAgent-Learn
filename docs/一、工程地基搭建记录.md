## 一、项目目录结构

在 `D:/desktop/guashuai-project` 下创建功能目录：`backend/`（FastAPI + NiceGUI，统一Python进程）、`agents/`（LangGraph多智能体工作流）、`rag/`（向量检索）、`db/`（数据库模型）、`tests/`（测试）、`deploy/`（Docker配置）、`scripts/`（启停脚本）、`docs/`（本地文档，不提交）。各Python模块目录含 `__init__.py`。

***

## 二、Git版本控制与GitHub仓库

**初始化**：`git init`，创建 `.gitignore`，过滤 `__pycache__`、`node_modules`、`.env`、`chroma_db/`、`docs/` 等。

**SSH认证**：`gh` CLI和HTTPS交互式认证均不可用，生成ED25519 SSH密钥对，公钥添加至GitHub Settings → SSH Keys，配置 `~/.ssh/config` 指定IdentityFile。

**障碍修复**：全局git配置中存在 `ghproxy.com` 代理转发规则（`url.insteadof`），导致连接超时。`git config --global --unset-all` 移除后恢复直连。

**仓库**：https://github.com/tpys11/CoAgent-Learn

## 三、技术选型对齐

**最终确认的技术栈**（与赛题概念设计对齐）：

| 层次    | 选型                     | 说明                                  |
| ----- | ---------------------- | ----------------------------------- |
| 全栈语言  | Python 3.12            | 统一技术栈                               |
| 后端框架  | FastAPI                | 高性能异步Web接口                          |
| 前端    | NiceGUI                | Python生态，WebSocket双向通信，内置于FastAPI之上 |
| 多智能体  | LangGraph              | 控制逻辑强，适合复杂Agent工作流编排                |
| 工具调用  | Function Calling → MCP | 先成熟方案，后期升级标准化协议                     |
| 模型接入  | BaseLLM基类 → DeepSeek   | 统一封装对话/Function Calling/嵌入，扩展只需新增子类 |
| 业务数据库 | SQLite                 | 轻量，开发阶段最适合                          |
| 向量数据库 | Chroma                 | 学习难度低，开发速度快                         |
| 缓存    | Redis                  | 临时存储+限流+任务队列                        |
| 部署    | Git + Docker Compose   | 一键编排全部服务                            |

## 四、Python开发环境

`python -m venv .venv` 创建虚拟环境。`backend/requirements.txt` 使用 `>=` 宽松约束，pip安装12个核心依赖：FastAPI、NiceGUI、Uvicorn、LangGraph、LangChain、langchain-deepseek、ChromaDB、Redis、SQLAlchemy等。

`backend/main.py` 实现FastAPI应用骨架，lifespan启动时校验 `DEEPSEEK_API_KEY` 等环境变量，挂载CORS中间件，`/health` 路由可做容器存活探测。

***

## 五、Docker Compose三服务编排

使用Docker 29.4.1 + Compose v5.1.3，**3个服务**（NiceGUI合并到backend进程，无需独立前端容器）：

| 服务      | 镜像/构建                  | 端口   | 验证                            |
| ------- | ---------------------- | ---- | ----------------------------- |
| backend | Python 3.12-slim 构建    | 8000 | `/health` → `{"status":"ok"}` |
| redis   | redis:7-alpine         | 6379 | 运行中                           |
| chroma  | chromadb/chroma:latest | 8001 | 心跳正常                          |

compose文件位于 `deploy/` 子目录，`context` 和 `env_file` 使用 `../` 指回根目录。数据通过命名卷持久化，代码通过bind mount实现热更新。
