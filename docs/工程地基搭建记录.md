### 2.2 SSH密钥配置

由于环境中的 `gh` CLI不可用，改用SSH方式连接GitHub：
1. 执行 `ssh-keygen -t ed25519` 生成ED25519密钥对
2. 将公钥添加到 GitHub Settings → SSH and GPG keys
3. 配置 `~/.ssh/config`，指定GitHub连接使用该密钥

## 三、Python开发环境配置

### 3.1 虚拟环境

使用Python 3.12内置 `venv` 模块创建虚拟环境于 `.venv/` 目录，与全局Python环境隔离，确保依赖版本不受系统其他项目影响。

### 3.2 核心依赖安装

后端依赖通过 `pip install -r requirements.txt` 完成安装。初始使用精确版本号（`==`）导致依赖解析长时间回溯，改为最低版本约束（`>=`）后顺利安装。已安装核心包：

- **Web框架**: FastAPI 0.139.2、Uvicorn 0.51.0
- **多智能体调度**: LangGraph 1.2.9、LangChain 1.3.14
- **模型接入**: langchain-deepseek 1.1.0（OpenAI SDK 2.46.0）
- **向量数据库**: ChromaDB 1.5.9
- **缓存与存储**: Redis 7.0.1、SQLAlchemy 2.0.44
- **工具库**: python-dotenv、Pydantic、httpx 等

### 3.3 FastAPI应用骨架

`backend/main.py` 实现了最简可运行后端：

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from dotenv import load_dotenv

load_dotenv()

app = FastAPI(
    title="CoAgent-Learn",
    description="领域知识个性化生成与多智能体协同决策系统",
    version="0.1.0",
)

app.add_middleware(CORSMiddleware, allow_origins=["*"], ...)

@app.get("/health")
async def health_check():
    return {"status": "ok", "version": "0.1.0"}
```

**关键设计点**：
- `lifespan` 启动时自动校验必需环境变量（`DEEPSEEK_API_KEY` 等），缺失则抛出明确错误
- CORS中间件允许跨域请求，为前后端分离做好准备
- `/health` 健康检查接口可用于后续Docker容器存活探测
