# 三（1）、多智能体系统与展示

## 一、Agent 职责一览

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| 🎯 调度 Agent | 编排决策：判断信息是否足够，选择调用哪个子 Agent | 用户输入+已有信息 | 调用指令 / "信息足够" |
| 📥 输入信息处理 Agent | 格式识别与转化：PDF→opendataloader，非PDF→markitdown | 原始用户输入 | 结构化文本 |
| 🔍 学情诊断 Agent | 分析用户知识水平，生成诊断画像 | 用户问题 | 知识画像 JSON |
| 📚 知识库管理 Agent | 从 Chroma 向量库检索相关知识片段 | 检索查询 | 相关知识片段列表 |
| 🔎 搜索 Agent | 联网搜索补充信息（SearXNG+Perplexica） | 搜索查询 | 搜索结果列表 |
| 🧠 记忆管理 Agent | 读写用户记忆（L1/L2/L3三层） | 记忆读写指令 | 记忆数据 |
| 📤 信息整理与生成 Agent | 将调度汇总的信息整理为学习内容 | 多源信息 | 生成内容初版 |
| ⚖️ 审核裁判 Agent | 交叉验证内容准确性，标注可疑点 | 生成内容 | 审核报告（通过/不通过） |

## 二、按流程展开

### 1、后台逻辑（LangGraph 实现）

**输入阶段**：信息输入Agent接管 → 输出传递给调度Agent。

**核心逻辑阶段**：调度Agent用 LLM 自主判断是否需要调子Agent（诊学情/知识库/搜索/记忆），每个子Agent完成后回到调度，调度继续判断，直到信息足够。

**输出阶段**：信息整理与生成→审核裁判→通过则微调输出，不通过则重试（最多3次）。

**文件**：`agents/graph.py`（工作流编排）、`agents/prompts.py`（提示词定义）、`backend/main.py`（SSE流式接口）

### 2、通信模式

A2A（Agent 调用 Agent）：所有子 Agent 只能与调度 Agent 通信，子 Agent 之间不直接交互。调度单向调用子Agent并收回结果。

### 3、前端展示

- **画布**：React Flow (@xyflow/react)，8 个磨砂节点 + 黑色箭头连线。Agent 执行时节点放大高亮，执行过的节点保留显示
- **思考过程**：流式 SSE 推送，逐 token 实时展示在灰色气泡中。每个 Agent 调用 `think_then_json` 方法，先用自然语言输出思考过程，再用 ```json``` 输出结果。思考完成后气泡自动折叠，回复消息底部可展开查看
- **控制栏设置**：检索模式/输出形式/输出内容/输入优化 — 通过 POST body 传给后端，注入到对应 Agent 的 System Prompt

## 三、单 Agent 的控制逻辑

### 1、全局性提示词

- **隐藏部分**：用户在控制栏选择的模式（检索/输出/输入优化），转为文本填充到对应 Agent 的 System Prompt
- **开放部分**：用户在 Agent 配置面板自定义的提示词，保存到前端状态，每次请求随 settings 传给后端

### 2、Skill 实现

每个 Agent 的 `think_then_json` 方法流程：
1. 调用 DeepSeek `chat_stream`（流式），每收到 token 通过回调推送到 SSE
2. 收集完整输出，用正则提取 ```json``` 代码块作为结构化结果
3. JSON 之前的部分作为自然语言思考链
4. 思考链通过 SSE 逐 token 推送到前端实时展示
