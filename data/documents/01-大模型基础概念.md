# 大语言模型（LLM）基础概念

## 什么是大语言模型

大语言模型是一种通过海量文本数据训练出来的神经网络模型，能够理解和生成人类语言。它最核心的能力是"预测下一个词"——给定前面的文字，模型会计算接下来最可能出现的词是什么。

你日常使用的 ChatGPT、DeepSeek、Claude 等对话产品，底层都是大语言模型在工作。

## 核心概念：Token

大模型不是按"字"或"词"来理解文本的，而是按 Token。一个 Token 大约等于：

- 英文：约 1 个词或 3/4 个单词
- 中文：约 1 个汉字或 1 个常见词组

比如"人工智能改变世界"会被切分为 人工智能 / 改变 / 世界，约 3 个 Token。

Token 非常重要，因为：
1. 模型的输入输出都按 Token 计费
2. 每个模型有最大 Token 限制（称为上下文窗口）
3. 超出限制的内容会被截断

## 核心概念：上下文窗口

上下文窗口是指模型一次能"看到"的最大 Token 总量（输入+输出）。

早期模型只有 2K-4K 的上下文窗口，现在的模型通常支持 128K 甚至 1M 的上下文窗口。更大的窗口意味着模型可以一次处理更长的文档或更长的对话历史。

但要注意：上下文窗口越大，处理成本越高，速度越慢。

## 核心概念：温度（Temperature）

温度控制模型输出的"随机性"，取值范围通常是 0 到 2：

- 温度低（0-0.3）：输出更确定、一致，适合需要精确答案的场景
- 温度中（0.5-0.8）：创造性与准确性的平衡
- 温度高（0.9-2.0）：输出更多样、更有创意的回答，但可能不准确

对于学习和教育场景，建议使用中等偏低的温度（0.3-0.7），确保内容准确。

## 模型为什么会产生"幻觉"

大模型本质上是"预测下一个词"的统计机器，它并不真正"理解"事实。当模型被问到它训练数据中不包含的信息时，它会根据概率来"编造"看起来合理的答案——这就是"幻觉"。

这也是为什么本系统要用 RAG（检索增强生成）技术——让模型基于真实文档回答问题，而不是依赖它可能过时或不准确的内部记忆。
