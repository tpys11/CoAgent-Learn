# Prompt Engineering（提示词工程）

## 什么是 Prompt

Prompt 就是你发给大模型的指令。好的 Prompt 能让模型输出精准、有用的回答，差的 Prompt 可能得到答非所问的结果。

## 写好 Prompt 的基本原则

### 1. 明确角色
告诉模型"你是谁"，它会更精准地调整输出风格和知识域。

**示例**：你是人工智能领域的资深讲师。请用通俗易懂的语言向初学者解释什么是RAG。

### 2. 明确任务
用清晰的动词描述你想要的输出。

**示例**：请列出RAG技术的5个核心步骤，每个步骤用一句话解释。

### 3. 指定格式
告诉模型你期望什么格式的输出。

**示例**：请用Markdown格式回答，每个步骤作为二级标题，配2-3句说明。

### 4. 给出约束
限制回答的长度、范围、风格。

**示例**：总字数不超过300字，不要使用专业术语，面向完全的AI初学者。

### 5. 提供示例（Few-shot）
给模型看一两个你期望的输出样例，它会更好地理解你的需求。

## 结构化 Prompt 模板

一个完整的结构化 Prompt 通常包含：

```
角色：你是XXX
任务：请做XXX
格式：输出格式是XXX
约束：字数限制XXX，不要XXX
示例：（可选）比如像这样：XXX
```

## 常见错误

- 指令过于模糊（"给我讲讲AI"——太宽泛）
- 多个任务混在一个Prompt里（先让模型做A再做B再总结——拆成多次对话）
- 缺少输出格式要求（模型可能给出不适合直接使用的格式）
- 前后矛盾（告诉模型"简洁"同时又要求"详细解释"）

## 实操建议

刚开始使用AI时，可以先写一个简单的Prompt，看输出，然后逐步优化。这个过程本身就能帮你更好地理解AI的工作方式。
