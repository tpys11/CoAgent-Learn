# AI Agent（智能体）基础

## 什么是AI Agent

AI Agent 是一个能够自主完成任务的智能程序。它与普通的"聊天AI"关键区别在于：

- 聊天AI：用户问 → AI答 → 结束
- AI Agent：用户给目标 → AI规划步骤 → 调用工具 → 观察结果 → 调整 → 继续执行 → 直到完成目标

Agent = 大模型（大脑）+ 工具（手和脚）+ 记忆（笔记）+ 规划（思维能力）

## Agent 的四种核心设计模式

来自吴恩达课程的系统总结：

### 1. Reflection（反思）
Agent 检查自己上一步的输出，发现错误或不足，然后改进。就像学生做完题后自己检查一遍。

### 2. Tool Use（工具调用）
Agent 调用外部工具获取信息或执行动作。例如：搜索网页、查询数据库、读写文件、发送API请求、运行代码等。

Function Calling 是目前最成熟的工具调用方案——大模型输出一个JSON，描述它想调用哪个函数以及什么参数，后台代码执行这个函数，把结果返回给模型。

### 3. Planning（规划）
Agent 把复杂任务拆解为多个步骤，按顺序执行，动态调整。比如"写一篇关于AI Agent的教程"，Agent 会先规划：搜索资料 → 列出大纲 → 逐章写作 → 检查错别字 → 最终排版。

### 4. Multi-Agent（多Agent协作）
多个专业Agent分工合作，每个Agent负责自己擅长的部分，通过通信机制协作完成复杂任务。

## Agent Loop（智能体循环）

一个标准的Agent工作循环：

1. Think（思考）：分析当前状态和目标，决定下一步做什么
2. Act（行动）：执行决策——调用工具或生成回答
3. Observe（观察）：查看工具返回的结果
4. 重复：回到 Think，直到任务完成

这个雷Act循环（Think-Act-Observe）是绝大多数Agent系统的核心骨架。
