| frontend | node:22-alpine 构建      | 5173 | React 前端    |

## 五、Python 开发环境

`python -m venv .venv` 虚拟环境。`backend/requirements.txt` 含 11 个核心依赖：FastAPI、Uvicorn、LangGraph、LangChain、langchain-deepseek、ChromaDB、Redis、SQLAlchemy、python-dotenv、pydantic、httpx。`backend/main.py` 实现纯 API 骨架，`/health` 存活探测，`POST /api/chat` 对话接口。

## 六、BaseLLM — 大模型统一调用层

在 `backend/core/` 下创建统一的模型调用层，让上层 Agent 代码无需关心底层模型差异。

### 核心接口

**`chat(messages)`** — 普通对话。内部自动：①过滤 DeepSeek R1 思考标签 ②超时重试（最多3次，间隔2s→4s→6s递增）③记录 token 消耗。

**`chat_with_json(messages, output_schema)`** — JSON 结构化输出（Agent 间传数据用）。自动注入 Schema 指令 + `response_format={"type": "json_object"}` + 两层容错解析。

### DeepSeekLLM 子类

仅 6 行代码。新增模型只需写类似子类，Agent 层代码不改。

**技术选型理由**：使用 OpenAI SDK，DeepSeek/GLM/Qwen/Kimi 全部兼容 OpenAI chat/completions 接口，避免 langchain 版本冲突。
